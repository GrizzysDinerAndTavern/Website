# Grizzy’s Diner & Tavern Website

Static one-page website for Grizzy’s Diner & Tavern, prepared for **GitHub Pages**.

## Required repository structure

Upload the contents of this folder directly to the top level of your GitHub repository:

```text
index.html
README.md
.nojekyll
assets/
  breakfast-menu.png
  grizzys-logo.png
  lunch-menu-front.png
  lunch-menu-back.png
```

`index.html` and the `assets` folder must be beside each other at the repository root. Do not upload only the ZIP file, and do not place these files inside an extra nested folder.

## Publish with GitHub Pages

1. Open the GitHub repository.
2. Go to **Settings**.
3. Select **Pages** in the left sidebar.
4. Under **Build and deployment**, set **Source** to **Deploy from a branch**.
5. Choose the **main** branch.
6. Choose the **/(root)** folder.
7. Click **Save**.

GitHub will publish the website at a URL similar to:

```text
https://YOUR-USERNAME.github.io/YOUR-REPOSITORY-NAME/
```

The first deployment may take several minutes. The exact live URL will appear in **Settings → Pages**.

## Updating the website

Replace or edit files in the repository and commit the changes. GitHub Pages will automatically redeploy the site.

Keep these image filenames unchanged unless you also update their paths inside `index.html`:

- `assets/grizzys-logo.png`
- `assets/breakfast-menu.png`
- `assets/lunch-menu-front.png`
- `assets/lunch-menu-back.png`
